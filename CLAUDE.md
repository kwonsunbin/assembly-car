Going to do Refactoring from the Agentic Engineering point of view.

Main Mission : Legacy System의 차량 조립 코드를 분석하고, 공학적 관점(비판적 사고)으로 검토하여 더 좋은 구조로 리팩토링을 수행한다.
Rule : (Human-in-the-loop) AI가 독단적으로 코드를 수정하게 하지 않고, 변경 전에 요구사항과 설계(기획)를 사람과 먼저 합의한 후 구현 및 검증을 반복한다.

4. 리팩토링 지시사항 (Claude의 임무)
4.1. 코드 분석 및 문제점 도출: 위 레거시 코드에서 발생하는 공학적 문제점(예: 강한 결합도, if-else 분기문의 중복, 확장 불가능한 구조, OCP 위반 등)을 먼저 비판적으로 분석해 줘.
4.2. 단계별 계획 수립: 한 번에 코드를 다 갈아엎지 말고, 객체지향 설계 원칙(SOLID)을 적용하여 안전하게 구조를 변경할 수 있는 단계별 리팩토링 계획(PLAN.md 형태)을 수립해 줘.
4.3. 리팩토링 실행: 합리적인 구조(예: 다형성 활용, 전략 패턴 또는 상태 패턴 적용, 차량과 부품의 객체 분리 등)로 개선된 코드를 제시해 줘.
